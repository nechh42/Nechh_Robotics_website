Overview

The Nechh Robotics REST API provides programmatic access to trading signals, historical performance data, and asset information. All requests use HTTPS and return JSON.

Base URL:

https://api.nechhrobotics.com/v1

Authentication

All API requests require an API key passed in the request header:

# Request header Authorization: Bearer nr_live_xxxxxxxxxxxxxxxxxxxxxxxxxx

Your API key is available in your subscriber dashboard. Keep it secret — treat it like a password. Do not commit it to public repositories.

Rate Limits

Rate limit headers are included in every response: X-RateLimit-Remaining and X-RateLimit-Reset.

GET /signals

Returns a list of recent trading signals.

GET /v1/signals
Retrieve signals with optional filters.

Query Parameters

ParameterTypeDescription
assetstringFilter by asset (e.g. BTCUSDT)
directionstringLONG or SHORT
statusstringopen, closed, all (default: all)
limitintegerResults per page (max 100, default 20)
fromISO8601Start timestamp filter
toISO8601End timestamp filter

Example Request

curl -X GET "https://api.nechhrobotics.com/v1/signals?asset=ETHUSDT&limit=5" \ -H "Authorization: Bearer nr_live_xxxx"

Example Response

{ "data": [ { "id": "sig_a1b2c3d4", "asset": "ETHUSDT", "direction": "LONG", "entry": 3142.50, "stop_loss": 3096.80, "take_profit": 3198.20, "strategy": "EMA_PULLBACK", "regime": "TRENDING", "status": "closed", "outcome": "WIN", "pnl_pct": 1.77, "created_at": "2026-05-01T14:00:00Z", "closed_at": "2026-05-01T18:22:00Z" } ], "meta": { "total": 148, "page": 1, "limit": 5 } }

GET /performance

GET /v1/performance
Aggregate performance statistics for a given period.

Query Parameters

ParameterTypeDescription
periodstring7d, 30d, 90d, 1y, all
assetstringFilter by specific asset

Example Response

{ "period": "30d", "total_signals": 18, "wins": 10, "losses": 8, "win_rate": 55.6, "profit_factor": 1.87, "avg_win_pct": 1.72, "avg_loss_pct": -0.92, "max_drawdown_pct": 4.2 }

GET /assets

GET /v1/assets

Webhook Setup

Receive real-time signal notifications via HTTP POST to your endpoint. Configure in your subscriber dashboard.

Webhook Events

EventTrigger
signal.createdNew signal generated
signal.closedSignal outcome determined (win/loss/expired)

Webhook Payload Example

{ "event": "signal.created", "timestamp": "2026-05-01T14:00:12Z", "data": { /* Signal Object */ } }
